# Topics Index

Este índice separa el conocimiento por tema y por uso.

## Canonical topics

- `image-generation`: cómo diseñar prompts, variantes, validaciones y salidas.
- `transposition`: cómo mover material musical sin perder intención.
- `knowledge-curation`: cómo convertir trabajo real en memoria reutilizable.
- `release-gates`: criterios para decidir si algo está listo para uso estable.
- `project-patterns`: patrones repetibles extraídos de proyectos BAGO.
- `learned-lessons`: registro de lecciones concretas promovidas al canon.
- `sync-protocol`: cómo mantener el árbol local compatible con GitHub.
- `publication-contract`: cómo publicar el runtime con o sin knowledge.
- `engine-contract`: cómo mantener el motor limpio y refrescable durante el desarrollo.

## Regla de lectura

Primero el índice. Después el tema. Después el ejemplo. Al final el manifiesto
si hace falta validar el contrato.
